# Images Directory

This directory is for storing static image assets used throughout the Fealess application.

## Current Status

**No images are currently in the application.** This directory structure is prepared for future image additions.

## Directory Structure

Organize images in subdirectories by purpose:

```
images/
├── hero/          # Hero section background images
├── leaders/       # Leadership profile photos
├── services/      # Service-related images and icons
├── values/        # Value-related images and icons
├── icons/         # General purpose icons
└── content/       # General content images
```

## Adding Images

When adding images to this directory:

1. **Optimize images first**:
   - Compress images to reduce file size
   - Use appropriate dimensions (don't upload 4K for 400px display)
   - Target: <200KB for hero images, <50KB for thumbnails

2. **Use appropriate formats**:
   - JPEG for photographs
   - PNG for images with transparency
   - SVG for icons and logos
   - WebP/AVIF are automatically generated by Next.js

3. **Follow naming conventions**:
   - Use lowercase with hyphens: `hero-background.jpg`
   - Be descriptive: `bill-clinton-profile.jpg` not `img1.jpg`
   - Include dimensions if multiple sizes: `logo-small.svg`, `logo-large.svg`

4. **Update components**:
   - Use `next/image` component (see `IMAGE_OPTIMIZATION_GUIDE.md`)
   - Add descriptive alt text for accessibility
   - Configure responsive sizes for large images

## Image Requirements

All images must meet these requirements:

- ✅ Used with `next/image` component (not `<img>` tag)
- ✅ Include descriptive alt text (Requirement 11.5)
- ✅ Optimized and compressed before adding
- ✅ Organized in appropriate subdirectory
- ✅ Tested on multiple devices and screen sizes

## Configuration

Image optimization is configured in `next.config.js`:
- Formats: AVIF, WebP (automatic)
- Device sizes: 640px to 3840px
- Image sizes: 16px to 384px
- Cache TTL: 60 seconds
