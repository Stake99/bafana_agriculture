# Next.js Image Optimization Guide

## Overview

This application is configured to use Next.js Image optimization for all images. The `next/image` component provides automatic image optimization, lazy loading, and responsive image serving.

## Configuration

Image optimization is configured in `next.config.js`:

```javascript
images: {
  formats: ['image/avif', 'image/webp'],
  deviceSizes: [640, 750, 828, 1080, 1200, 1920, 2048, 3840],
  imageSizes: [16, 32, 48, 64, 96, 128, 256, 384],
  minimumCacheTTL: 60,
}
```

### Configuration Details

- **formats**: Images are automatically served in AVIF and WebP formats when supported by the browser, with fallback to original format
- **deviceSizes**: Breakpoints for responsive images matching common device widths
- **imageSizes**: Sizes for smaller images (icons, thumbnails, etc.)
- **minimumCacheTTL**: Cache optimized images for at least 60 seconds

## Usage Guidelines

### Basic Image Usage

Always use the `next/image` component instead of the HTML `<img>` tag:

```tsx
import Image from 'next/image';

export default function Component() {
  return (
    <Image
      src="/images/example.jpg"
      alt="Descriptive alt text for accessibility"
      width={800}
      height={600}
      priority={false}
    />
  );
}
```

### Required Props

1. **src**: Path to the image (relative to `public/` directory or external URL)
2. **alt**: Descriptive text for accessibility (REQUIRED for all images)
3. **width** and **height**: Dimensions in pixels (required for static images)

### Optional Props

- **priority**: Set to `true` for above-the-fold images (e.g., hero images) to disable lazy loading
- **quality**: Image quality (1-100, default is 75)
- **placeholder**: Show blur placeholder while loading (`"blur"` or `"empty"`)
- **sizes**: Responsive sizes for different viewports

### Hero Section Images

For large hero images that appear above the fold:

```tsx
<Image
  src="/images/hero-background.jpg"
  alt="Sustainable livestock farming landscape"
  width={1920}
  height={1080}
  priority={true}
  quality={85}
  className="w-full h-auto"
/>
```

### Leadership Profile Images

For profile images in the Leadership section:

```tsx
<Image

  alt="Bill Clinton, CEO of Fealess Company"
  width={400}
  height={400}
  className="rounded-full"
  sizes="(max-width: 768px) 200px, 400px"

```

### Service/Value Icons

For smaller icons or decorative images:

```tsx

  alt="Livestock consultation icon"
  width={64}
  height={64}
/>
```

## Accessibility Requirements

### Alt Text Guidelines

All images MUST include descriptive alt text:

- **Informative images**: Describe the content and function
  - Good: `alt="Sustainable livestock grazing on pasture"`
  - Bad: `alt="image1"` or `alt=""`

- **Decorative images**: Use empty alt text if purely decorative
  - `alt=""` (empty string, not omitted)

- **Profile images**: Include person's name and role
  - `alt="Beatrice Williams, AgriTech Entrepreneur"`

- **Icons with adjacent text**: Can use empty alt if text conveys meaning
  - `alt=""` when icon is next to "Sustainable Livestock Consultation"

### WCAG Compliance

- All images must have alt attributes (Requirement 11.5)
- Alt text should be concise but descriptive
- Avoid phrases like "image of" or "picture of"
- For complex images (charts, diagrams), provide detailed descriptions

## Responsive Images

Use the `sizes` prop to optimize image loading for different viewports:

```tsx
<Image
  src="/images/example.jpg"
  alt="Description"
  width={1200}
  height={800}
  sizes="(max-width: 640px) 100vw, (max-width: 1024px) 50vw, 33vw"
/>
```

This tells the browser:
- On mobile (≤640px): image takes full viewport width
- On tablet (≤1024px): image takes 50% of viewport width
- On desktop (>1024px): image takes 33% of viewport width

## Performance Best Practices

1. **Use appropriate formats**:
   - JPEG for photographs
   - PNG for images with transparency
   - SVG for icons and logos
   - WebP/AVIF are automatically generated by Next.js

2. **Optimize source images**:
   - Compress images before adding to project
   - Use appropriate dimensions (don't upload 4K images for 400px display)
   - Target file sizes: <200KB for hero images, <50KB for thumbnails

3. **Lazy loading**:
   - Images below the fold are automatically lazy-loaded
   - Set `priority={true}` only for above-the-fold images

4. **Blur placeholders**:
   - For local images, Next.js can generate blur placeholders automatically
   - Improves perceived performance during image loading

## External Images

To use images from external domains, configure allowed domains in `next.config.js`:

```javascript
images: {
  remotePatterns: [
    {
      protocol: 'https',
      hostname: 'example.com',
      pathname: '/images/**',
    },
  ],
}
```

## Example: Adding a Hero Background Image

When ready to add a hero background image:

1. Add image to `public/images/hero/background.jpg`
2. Update `components/Hero.tsx`:

```tsx
import Image from 'next/image';

export default function Hero({ headline, rating, subheading }: HeroProps) {
  return (
    <section id="hero" className="relative min-h-screen">
      <Image
        src="/images/hero/background.jpg"
        alt="Sustainable animal agriculture landscape with livestock"
        fill
        priority={true}
        quality={85}
        className="object-cover"
      />
      <div className="relative z-10">
        {/* Existing hero content */}
      </div>
    </section>
  );
}
```

3. Verify alt text is descriptive and meaningful
4. Test on multiple devices and screen sizes
